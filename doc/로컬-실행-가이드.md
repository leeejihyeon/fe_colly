# 로컬 실행 가이드

## 1. 목적
- `collyapp`를 로컬에서 안정적으로 실행하기 위한 표준 절차를 정의한다.
- 인증 설정값까지 포함해 실제 로그인 플로우를 검수 가능한 상태를 목표로 한다.

## 2. 사전 준비
### 공통
- Node.js 설치
- npm 설치
- 프로젝트 경로: `/Users/lee/IdeaProjects/collyapp`
- 백엔드 API 실행 주소 기본값
  - iOS: `http://localhost:18080`
  - Android Emulator: `http://10.0.2.2:18080`

### iOS
- Xcode 설치 (App Store)
- iOS Simulator 컴포넌트 설치
- CocoaPods 설치
- Apple 개발자 계정 로그인

### Android
- Android Studio 설치
- Android SDK / Platform-Tools / Build-Tools / Emulator 설치
- Android Virtual Device(AVD) 1개 이상 생성

## 3. 최초 1회 설정
### 3.1 iOS 개발 도구 경로 설정
```bash
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
sudo xcodebuild -license accept
xcodebuild -version
```

### 3.2 iOS Pod 설치
```bash
cd /Users/lee/IdeaProjects/collyapp/ios
bundle exec pod install
cd ..
```

## 4. 인증 설정 파일 준비
### 4.1 iOS
1. `ios/Config/Local/CollyAuth.debug.local.xcconfig.example`를 참고해 `ios/Config/Local/CollyAuth.debug.local.xcconfig` 생성
2. 필요 시 release용으로 `ios/Config/Local/CollyAuth.release.local.xcconfig` 생성
3. Google Cloud Console에서 받은 값을 입력
   - `COLLY_GOOGLE_WEB_CLIENT_ID`
   - `COLLY_GOOGLE_IOS_CLIENT_ID`
   - `COLLY_GOOGLE_REVERSED_CLIENT_ID`
4. 실제 `GoogleService-Info.plist`를 Google Console에서 내려받아 `ios/collyapp/GoogleService-Info.plist`로 추가
5. Xcode에서 `GoogleService-Info.plist` Target Membership이 `collyapp`인지 확인
6. Xcode Target > Signing & Capabilities에서 `Sign in with Apple` capability가 활성화되어 있는지 확인

### 4.2 Android
1. `android/colly.auth.local.properties.example`를 참고해 `android/colly.auth.local.properties` 생성
2. 다음 값을 입력
   - `COLLY_APP_ENV`
   - `COLLY_API_BASE_URL`
   - `COLLY_GOOGLE_WEB_CLIENT_ID`
3. Google Cloud Console에 `com.collyapp` 패키지 기준 SHA-1 / SHA-256 등록
   - debug keystore와 release keystore를 분리 관리

## 6. 설정 주입 규칙
- JS 레이어는 `src/shared/config/env.ts`만 읽는다.
- iOS 값 출처
  - `ios/Config/*.xcconfig`
  - `ios/collyapp/Info.plist`
- Android 값 출처
  - `android/colly.auth.local.properties`
  - Gradle `BuildConfig`
- 값이 비어 있으면 로그인 버튼은 보이지만, 실제 로그인 시 상단 토스트로 설정 누락이 안내된다.

## 7. 실서비스 인증 체크리스트
- iOS bundle id: `com.collyapp`
- Android applicationId: `com.collyapp`
- 서버 `COLLY_AUTH_GOOGLE_AUDIENCE` = Google Web Client ID
- 서버 `COLLY_AUTH_APPLE_AUDIENCE` = `com.collyapp`
- iOS `COLLY_GOOGLE_REVERSED_CLIENT_ID` = `GoogleService-Info.plist`의 `REVERSED_CLIENT_ID`
- Apple Developer App ID에 `Sign in with Apple` 활성화
- Google Console에 iOS / Android 앱, Web client 모두 연결

## 8. 수동 검수 루틴
1. 비로그인 상태로 Community 진입
2. `Create Post` 눌러 로그인 모달 노출 확인
3. iOS에서 매직링크 메일 버튼으로 앱 복귀 확인
4. iOS에서 Google 로그인 시작 및 앱 복귀 확인
5. iOS에서 Apple 로그인 시스템 시트 노출 확인
6. Android에서 Google 로그인 시작 및 앱 복귀 확인
7. 앱 재실행 후 세션 복구 확인
8. 로그아웃 / 전체 로그아웃 후 보호 액션 차단 확인

## 9. 일상 개발 루프
1. 터미널1: `npm start`
2. 터미널2: `npx react-native run-ios` 또는 `npx react-native run-android`
3. 코드 수정 후 자동 반영(Fast Refresh)
